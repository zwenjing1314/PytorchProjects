# 遇到的问题
## CrossEntropyLoss() 的计算过程
函数原型：
```python
criterion = nn.CrossEntropyLoss()
loss = criterion(outputs, labels)
```
输入数据格式：
outputs：网络输出，形状 (batch_size, num_classes) = (4, 10)
每行是 10 个类别的原始得分（logits），范围 (-∞, +∞)
labels：真实标签，形状 (batch_size,) = (4,)
每个元素是整数，范围 [0, 9]

计算步骤（逐层详解）：
假设一个 batch 有 4 张图片，网络输出和标签如下：
```python
# 示例数据
outputs = torch.tensor([
    [2.1, -0.5, 3.2, 1.0, -1.2, 0.8, 2.5, -0.3, 1.8, 0.2],  # 图片 1
    [0.5, 1.2, -0.8, 2.3, 0.1, -1.5, 0.9, 1.7, -0.2, 0.6],  # 图片 2
    [-1.0, 0.3, 1.5, -0.7, 2.8, 0.2, -1.2, 0.9, 1.1, -0.4], # 图片 3
    [1.8, 0.2, -1.1, 0.7, 1.3, -0.9, 2.6, 0.4, -0.5, 1.0]   # 图片 4
])  # shape: (4, 10)

labels = torch.tensor([2, 3, 4, 6])  # shape: (4,)
# 真实类别：图片 1=鸟 (2), 图片 2=猫 (3), 图片 3=鹿 (4), 图片 4=船 (6)
```

计算流程（以第 1 张图片为例）：
```text
步骤 1: Softmax 归一化
对 outputs[0] 的每个元素做指数运算并归一化：

原始得分：[2.1, -0.5, 3.2, 1.0, -1.2, 0.8, 2.5, -0.3, 1.8, 0.2]
        ↓ exp(x)
指数：[e^2.1, e^(-0.5), e^3.2, e^1.0, e^(-1.2), e^0.8, e^2.5, e^(-0.3), e^1.8, e^0.2]
     ≈ [8.17, 0.61, 24.53, 2.72, 0.30, 2.23, 12.18, 0.74, 6.05, 1.22]
        ↓ 求和 Σ ≈ 58.75
概率：[8.17/58.75, 0.61/58.75, 24.53/58.75, ...]
     ≈ [0.139, 0.010, 0.418, 0.046, 0.005, 0.038, 0.207, 0.013, 0.103, 0.021]
     ↑ 所有概率相加 = 1.0

步骤 2: 提取正确类别的概率
labels[0] = 2（鸟）
预测为"鸟"的概率 = 0.418

步骤 3: 计算负对数似然
loss[0] = -log(0.418) ≈ -(-0.872) ≈ 0.872

步骤 4: 对 batch 内所有样本求平均
loss = (loss[0] + loss[1] + loss[2] + loss[3]) / 4
```

数学公式：
```math
\text{CrossEntropyLoss} = -\frac{1}{N}\sum_{i=1}^{N}\log(p_{i,label})
```

$$$$
其中：
$N$ = batch_size
$p_{i,label}$ = 第 $i$ 个样本对应真实标签的预测概率
PyTorch 内部实现（等价代码）：

PyTorch 内部实现（等价代码）：
```python
# CrossEntropyLoss 等价于以下两步：
softmax = nn.Softmax(dim=1)
nll_loss = nn.NLLLoss()

# 方法 1: 分开计算
probabilities = softmax(outputs)  # (4, 10)
log_probabilities = torch.log(probabilities)  # (4, 10)
loss = nll_loss(log_probabilities, labels)  # 标量

# 方法 2: 直接用 CrossEntropyLoss（更高效）
criterion = nn.CrossEntropyLoss()
loss = criterion(outputs, labels)  # 一步完成，数值更稳定
```

## optim.SGD() 的参数和操作
函数原型：
```python
optimizer = optim.SGD(net.parameters(), lr=0.001, momentum=0.9)
```
net.parameters() 要优化的参数 告诉优化器需要更新哪些权重（w）和偏置（b）

输入数据来源：
```text
optimizer 接收的输入：
└─ net.parameters() 包含：
   ├─ conv1.weight: (6, 3, 5, 5)      # Conv1 层权重
   ├─ conv1.bias: (6,)                # Conv1 层偏置
   ├─ conv2.weight: (16, 6, 5, 5)     # Conv2 层权重
   ├─ conv2.bias: (16,)               # Conv2 层偏置
   ├─ fc1.weight: (120, 400)          # 全连接层 1 权重
   ├─ fc1.bias: (120,)                # 全连接层 1 偏置
   ├─ fc2.weight: (84, 120)           # 全连接层 2 权重
   ├─ fc2.bias: (84,)                 # 全连接层 2 偏置
   ├─ fc3.weight: (10, 84)            # 输出层权重
   └─ fc3.bias: (10,)                 # 输出层偏置
```

操作流程（一个完整的训练步骤）：
```python
# 步骤 0: 初始化
optimizer = optim.SGD(net.parameters(), lr=0.001, momentum=0.9)

for epoch in range(2):
    for i, data in enumerate(trainloader, 0):
        inputs, labels = data
        
        # 步骤 1: 清零梯度
        optimizer.zero_grad()
        # 作用：清空上一步残留的梯度值
        # 原因：PyTorch 默认累加梯度，不清零会导致梯度爆炸
        
        # 步骤 2: 前向传播
        outputs = net(inputs)  # (4, 3, 32, 32) → (4, 10)
        
        # 步骤 3: 计算损失
        loss = criterion(outputs, labels)  # 标量
        
        # 步骤 4: 反向传播（计算梯度）
        loss.backward()
        # 自动计算 ∂loss/∂w 和 ∂loss/∂b
        # 结果存储在每个参数的 .grad 属性中
        
        # 步骤 5: 更新参数
        optimizer.step()
        # 使用 SGD + Momentum 公式更新参数：
        # v = momentum * v_prev + lr * gradient
        # w = w - v
```
SGD with Momentum 的更新公式：
```python
# 伪代码展示 optimizer.step() 的内部逻辑：

for param in net.parameters():
    # param.grad 包含当前梯度
    
    # 动量项（速度）
    velocity = momentum * previous_velocity + gradient
    
    # 参数更新
    param.data = param.data - lr * velocity
    
    # 保存当前速度供下次迭代使用
    previous_velocity = velocity
```
数据流动完整示意图：
```text
输入数据流：
inputs (4, 3, 32, 32)
   ↓ forward()
outputs (4, 10)
   ↓ CrossEntropyLoss()
loss (标量)
   ↓ backward()
gradients {∂loss/∂w₁, ∂loss/∂b₁, ..., ∂loss/∂wₙ, ∂loss/∂bₙ}
   ↓ step()
更新后的参数 {w₁', b₁', ..., wₙ', bₙ'}
   ↓ 下一轮迭代
新的 outputs → 新的 loss → ...
```

不同优化器的对比：
```python
# 基础 SGD（无动量）
optim.SGD(net.parameters(), lr=0.01)

# SGD + Momentum（当前使用）
optim.SGD(net.parameters(), lr=0.001, momentum=0.9)

# Adam（自适应学习率，通常更快）
optim.Adam(net.parameters(), lr=0.001)

# RMSprop（适合 RNN）
optim.RMSprop(net.parameters(), lr=0.01)
```
