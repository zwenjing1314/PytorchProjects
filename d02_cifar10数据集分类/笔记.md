# 数据集介绍
CIFAR10数据集共有60000个样本，每个样本都是一张32*32像素的RGB图像（彩色图像），每个RGB图像又必定分为3个通道（R通道、G通道、B通道）。这60000个样本被分成了50000个训练样本和10000个测试样本。
CIFAR10数据集是用来监督学习训练的，那么每个样本就一定都配备了一个标签值（用来区分这个样本是什么），不同类别的物体用不同的标签值，CIFAR10中有10类物体，标签值分别按照0~9来区分,他们分别是飞机（ airplane ）、汽车（ automobile ）、鸟（ bird ）、猫（ cat ）、鹿（ deer ）、狗（ dog ）、青蛙（ frog ）、马（ horse ）、船（ ship ）和卡车（ truck ）。

CIFAR10数据集结构组成可分为这四个部分：
·train_x:(50000, 32, 32, 3)——训练样本
·train_y:(50000, 1)——训练样本标签
·test_x:(10000, 32, 32, 3)——测试样本
·test_y:(10000, 1)——测试样本标签

CIFAR-10数据集与MNIST数据集二者的区别主要是在以下几点：
·维度不同：CIFAR-10数据集有4个维度，MNIST数据集有3个维度
·图像类型不同：CIFAR-10数据集是RGB图像（有三个通道），MNIST数据集是灰度图像，这也是为什么CIFAR-10数据集比MNIST数据集多出一个维度的原因。
·图像内容不同：CIFAR-10数据集展示的是各种不同的物体（猫、狗、飞机、汽车…），MNIST数据集展示的是不同人的手写0~9数字。

## CrossEntropyLoss() 的计算过程
函数原型：
```python
criterion = nn.CrossEntropyLoss()
loss = criterion(outputs, labels)
```
输入数据格式：
outputs：网络输出，形状 (batch_size, num_classes) = (4, 10)
每行是 10 个类别的原始得分（logits），范围 (-∞, +∞)
labels：真实标签，形状 (batch_size,) = (4,)
每个元素是整数，范围 [0, 9]

计算步骤（逐层详解）：
假设一个 batch 有 4 张图片，网络输出和标签如下：
```python
# 示例数据
outputs = torch.tensor([
    [2.1, -0.5, 3.2, 1.0, -1.2, 0.8, 2.5, -0.3, 1.8, 0.2],  # 图片 1
    [0.5, 1.2, -0.8, 2.3, 0.1, -1.5, 0.9, 1.7, -0.2, 0.6],  # 图片 2
    [-1.0, 0.3, 1.5, -0.7, 2.8, 0.2, -1.2, 0.9, 1.1, -0.4], # 图片 3
    [1.8, 0.2, -1.1, 0.7, 1.3, -0.9, 2.6, 0.4, -0.5, 1.0]   # 图片 4
])  # shape: (4, 10)

labels = torch.tensor([2, 3, 4, 6])  # shape: (4,)
# 真实类别：图片 1=鸟 (2), 图片 2=猫 (3), 图片 3=鹿 (4), 图片 4=船 (6)
```

计算流程（以第 1 张图片为例）：
```text
步骤 1: Softmax 归一化
对 outputs[0] 的每个元素做指数运算并归一化：

原始得分：[2.1, -0.5, 3.2, 1.0, -1.2, 0.8, 2.5, -0.3, 1.8, 0.2]
        ↓ exp(x)
指数：[e^2.1, e^(-0.5), e^3.2, e^1.0, e^(-1.2), e^0.8, e^2.5, e^(-0.3), e^1.8, e^0.2]
     ≈ [8.17, 0.61, 24.53, 2.72, 0.30, 2.23, 12.18, 0.74, 6.05, 1.22]
        ↓ 求和 Σ ≈ 58.75
概率：[8.17/58.75, 0.61/58.75, 24.53/58.75, ...]
     ≈ [0.139, 0.010, 0.418, 0.046, 0.005, 0.038, 0.207, 0.013, 0.103, 0.021]
     ↑ 所有概率相加 = 1.0

步骤 2: 提取正确类别的概率
labels[0] = 2（鸟）
预测为"鸟"的概率 = 0.418

步骤 3: 计算负对数似然
loss[0] = -log(0.418) ≈ -(-0.872) ≈ 0.872

步骤 4: 对 batch 内所有样本求平均
loss = (loss[0] + loss[1] + loss[2] + loss[3]) / 4
```

数学公式：
```math
\text{CrossEntropyLoss} = -\frac{1}{N}\sum_{i=1}^{N}\log(p_{i,label})
```

$$$$
其中：
$N$ = batch_size
$p_{i,label}$ = 第 $i$ 个样本对应真实标签的预测概率
PyTorch 内部实现（等价代码）：

PyTorch 内部实现（等价代码）：
```python
# CrossEntropyLoss 等价于以下两步：
softmax = nn.Softmax(dim=1)
nll_loss = nn.NLLLoss()

# 方法 1: 分开计算
probabilities = softmax(outputs)  # (4, 10)
log_probabilities = torch.log(probabilities)  # (4, 10)
loss = nll_loss(log_probabilities, labels)  # 标量

# 方法 2: 直接用 CrossEntropyLoss（更高效）
criterion = nn.CrossEntropyLoss()
loss = criterion(outputs, labels)  # 一步完成，数值更稳定
```

## optim.SGD() 的参数和操作
函数原型：
```python
optimizer = optim.SGD(net.parameters(), lr=0.001, momentum=0.9)
```
net.parameters() 要优化的参数 告诉优化器需要更新哪些权重（w）和偏置（b）

输入数据来源：
```text
optimizer 接收的输入：
└─ net.parameters() 包含：
   ├─ conv1.weight: (6, 3, 5, 5)      # Conv1 层权重
   ├─ conv1.bias: (6,)                # Conv1 层偏置
   ├─ conv2.weight: (16, 6, 5, 5)     # Conv2 层权重
   ├─ conv2.bias: (16,)               # Conv2 层偏置
   ├─ fc1.weight: (120, 400)          # 全连接层 1 权重
   ├─ fc1.bias: (120,)                # 全连接层 1 偏置
   ├─ fc2.weight: (84, 120)           # 全连接层 2 权重
   ├─ fc2.bias: (84,)                 # 全连接层 2 偏置
   ├─ fc3.weight: (10, 84)            # 输出层权重
   └─ fc3.bias: (10,)                 # 输出层偏置
```

操作流程（一个完整的训练步骤）：
```python
# 步骤 0: 初始化
optimizer = optim.SGD(net.parameters(), lr=0.001, momentum=0.9)

for epoch in range(2):
    for i, data in enumerate(trainloader, 0):
        inputs, labels = data
        
        # 步骤 1: 清零梯度
        optimizer.zero_grad()
        # 作用：清空上一步残留的梯度值
        # 原因：PyTorch 默认累加梯度，不清零会导致梯度爆炸
        
        # 步骤 2: 前向传播
        outputs = net(inputs)  # (4, 3, 32, 32) → (4, 10)
        
        # 步骤 3: 计算损失
        loss = criterion(outputs, labels)  # 标量
        
        # 步骤 4: 反向传播（计算梯度）
        loss.backward()
        # 自动计算 ∂loss/∂w 和 ∂loss/∂b
        # 结果存储在每个参数的 .grad 属性中
        
        # 步骤 5: 更新参数
        optimizer.step()
        # 使用 SGD + Momentum 公式更新参数：
        # v = momentum * v_prev + lr * gradient
        # w = w - v
```
SGD with Momentum 的更新公式：
```python
# 伪代码展示 optimizer.step() 的内部逻辑：

for param in net.parameters():
    # param.grad 包含当前梯度
    
    # 动量项（速度）
    velocity = momentum * previous_velocity + gradient
    
    # 参数更新
    param.data = param.data - lr * velocity
    
    # 保存当前速度供下次迭代使用
    previous_velocity = velocity
```
数据流动完整示意图：
```text
输入数据流：
inputs (4, 3, 32, 32)
   ↓ forward()
outputs (4, 10)
   ↓ CrossEntropyLoss()
loss (标量)
   ↓ backward()
gradients {∂loss/∂w₁, ∂loss/∂b₁, ..., ∂loss/∂wₙ, ∂loss/∂bₙ}
   ↓ step()
更新后的参数 {w₁', b₁', ..., wₙ', bₙ'}
   ↓ 下一轮迭代
新的 outputs → 新的 loss → ...
```

不同优化器的对比：
```python
# 基础 SGD（无动量）
optim.SGD(net.parameters(), lr=0.01)

# SGD + Momentum（当前使用）
optim.SGD(net.parameters(), lr=0.001, momentum=0.9)

# Adam（自适应学习率，通常更快）
optim.Adam(net.parameters(), lr=0.001)

# RMSprop（适合 RNN）
optim.RMSprop(net.parameters(), lr=0.01)
```

## optim.SGD() 的参数和操作

### 参数解析：
```python
optim.SGD(
    params=net.parameters(),    # 需要优化的参数（所有层的权重和偏置）
    lr=0.001,                   # 学习率（Learning Rate）
    momentum=0.9,               # 动量因子
    weight_decay=0              # 权重衰减（L2 正则化，默认 0）
)
```

### 内部状态管理：
```text
# SGD 优化器为每个参数维护以下信息：
param_state = {
    'momentum_buffer': tensor   # 动量缓冲区（累积历史梯度）
}

# 对于你的网络，优化器管理：
├── conv1.weight (6, 3, 5, 5) → 450 个参数 + 动量缓冲
├── conv1.bias (6,) → 6 个参数 + 动量缓冲
├── conv2.weight (16, 6, 5, 5) → 2400 个参数 + 动量缓冲
├── conv2.bias (16,) → 16 个参数 + 动量缓冲
├── fc1.weight (120, 400) → 48,000 个参数 + 动量缓冲
├── fc1.bias (120,) → 120 个参数 + 动量缓冲
├── fc2.weight (84, 120) → 10,080 个参数 + 动量缓冲
├── fc2.bias (84,) → 84 个参数 + 动量缓冲
└── fc3.weight (10, 84) → 840 个参数 + 动量缓冲
    └─> 总计约 61,996 个可学习参数
```

### SGD with Momentum 的核心算法：
```python
# 对于每个参数 θ（以 conv1.weight 为例）：

# 1. 获取当前梯度和动量缓冲区
g = conv1.weight.grad              # 当前梯度
v = momentum_buffer[conv1.weight]  # 历史动量

# 2. 更新动量缓冲区（指数加权移动平均）
v = momentum * v + g
v = 0.9 * v + g  # 你的代码中 momentum=0.9

# 3. 使用动量更新参数
conv1.weight = conv1.weight - lr * v
conv1.weight = conv1.weight - 0.001 * v
```

### 带详细数据流的更新过程：
```text
初始状态（epoch=0, iteration=0）：
conv1.weight = 随机初始化值（均值为 0，方差很小）
momentum_buffer = 0
loss = 2.5（随机猜测，损失很大）

第一次 backward 后：
conv1.weight.grad = 
    [[[[ 0.02, -0.01, ...], ...], ...], ...]  # shape (6, 3, 5, 5)
    └─> 有些正有些负，指示参数应该增大还是减小

第一次 step() 更新：
# 计算新动量
v = 0.9 * 0 + conv1.weight.grad = conv1.weight.grad

# 更新参数
conv1.weight = conv1.weight - 0.001 * conv1.weight.grad
    └─> 梯度为正的位置：权重减小
    └─> 梯度为负的位置：权重增大

第二次 iteration：
新的梯度 g₂ 计算出来
v = 0.9 * v + g₂  # 累积了之前的更新方向
conv1.weight = conv1.weight - 0.001 * v

动量的物理意义：
┌─────────────────────────────────────────┐
│ 如果梯度方向一致（连续几次都指向同一方向）│
│ → 动量 v 会越来越大                      │
│ → 参数更新速度加快                       │
│                                          │
│ 如果梯度方向震荡（正负交替）             │
│ → 动量相互抵消                           │
│ → 参数更新变慢，避免振荡                 │
└─────────────────────────────────────────┘
```

### 有动量的 SGD：
```text
# SGD with Momentum
v = μ * v + g      # μ = 0.9 动量因子
θ = θ - lr * v

优势：
1. 加速收敛：累积同方向的梯度
2. 减少振荡：抵消反方向的梯度
3. 冲出局部最优：惯性帮助越过小的障碍

物理类比：
想象一个球从山上滚下：
- 重力加速度 = 梯度方向
- 球的速度 = 动量 v
- 摩擦力 = (1 - momentum)
- 球越滚越快 = 动量累积
```

### 完整的参数更新数据流图
```text
训练一次迭代的完整流程（batch_size=4）：

1. 数据加载
   inputs: (4, 3, 32, 32)
   labels: (4,)

2. 梯度清零
   optimizer.zero_grad()
   └─> 所有 param.grad = 0

3. 前向传播
   outputs = net(inputs)
   └─> outputs: (4, 10)  # 4 个样本，10 个类别的得分
   
4. 计算损失
   loss = CrossEntropyLoss(outputs, labels)
   └─> loss: 标量（例如 2.5）

5. 反向传播
   loss.backward()
   └─> 自动计算链式法则
       └─> 填充所有 param.grad:
           conv1.weight.grad: (6, 3, 5, 5)
           fc3.weight.grad: (10, 84)
           ...

6. 参数更新
   optimizer.step()
   └─> 对每个参数执行 SGD 更新：
       
       对于 conv1.weight:
       v = 0.9 * v_old + conv1.weight.grad
       conv1.weight = conv1.weight - 0.001 * v
       
       对于 fc3.weight:
       v = 0.9 * v_old + fc3.weight.grad
       fc3.weight = fc3.weight - 0.001 * v
       
       ...（共 61,996 个参数被更新）

7. 结果
   损失从 2.5 → 2.4（略有下降）
   网络参数发生了微小但正确的改变
   下一次预测会更准确一点
```

本质：torch.optim 实现了如何根据梯度更新参数的算法，它是连接"计算梯度"（backward）和"实际训练"（参数更新）的桥梁。没有优化器，神经网络就永远停留在随机初始化状态，无法从数据中学习。

## orch.nn.functional 库详解
torch.nn.functional（通常简写为 F）是 PyTorch 中的函数式神经网络接口，提供了各种无状态的神经网络操作函数。
### 核心特点：无状态函数
在代码中的应用（第 13、119、122、124、125 行）：
```python
import torch.nn.functional as F

# 在 forward 前向传播中使用
x = self.pool(F.relu(self.conv1(x)))  # 第 119 行
x = self.pool(F.relu(self.conv2(x)))  # 第 122 行
x = F.relu(self.fc1(x))               # 第 124 行
x = F.relu(self.fc2(x))               # 第 125 行
```
无状态 vs 有状态 的本质区别：
```python
# ╔══════════════════════════════════════════════════════════╗
# ║  F.relu() (函数式 - 无状态)                              ║
# ╠══════════════════════════════════════════════════════════╣
# ║  • 不需要实例化对象                                      ║
# ║  • 不保存任何参数或状态                                  ║
# ║  • 每次调用都是独立的纯函数                              ║
# ║  • 只完成计算，不占用额外内存                            ║
# ╚══════════════════════════════════════════════════════════╝

# ╔══════════════════════════════════════════════════════════╗
# ║  nn.ReLU() (模块式 - 有状态)                             ║
# ╠══════════════════════════════════════════════════════════╣
# ║  • 需要实例化：self.relu = nn.ReLU()                     ║
# ║  • 继承自 nn.Module，会注册到网络参数中                  ║
# ║  • 虽然 ReLU 本身没有参数，但仍然占用模块结构            ║
# ║  • 可以通过 .train() / .eval() 切换模式                  ║
# ╚══════════════════════════════════════════════════════════╝
```

### F.relu() 的详细数据流
 数据流向
 ```python
# 假设输入 x shape = (4, 3, 32, 32)  # batch_size=4

# Step 1: 卷积层（有状态，在 __init__ 中定义）
conv_output = self.conv1(x)
# conv1 = nn.Conv2d(3, 6, 5)
# 输出形状：(4, 6, 28, 28)
# 这个操作使用了 conv1.weight 和 conv1.bias 这两个可学习参数

# Step 2: ReLU 激活函数（无状态，直接调用）
relu_output = F.relu(conv_output)
# 输入形状：(4, 6, 28, 28)
# 输出形状：(4, 6, 28, 28)  # 形状不变

# ReLU 的逐元素计算：
对于 output 中的每个位置 (n, c, h, w):
    if conv_output[n, c, h, w] > 0:
        relu_output[n, c, h, w] = conv_output[n, c, h, w]
    else:
        relu_output[n, c, h, w] = 0

# 数学公式：ReLU(x) = max(0, x)

# 实际数值示例（简化为 1×1 的情况）：
conv_output = [[[-1.5, 2.3, -0.7, 1.2],
                [ 0.8,-0.4,  3.1,-1.0]]]
                
F.relu(conv_output) = [[[  0,  2.3,    0, 1.2],
                        [0.8,    0,  3.1,   0]]]

# Step 3: 池化层（有状态，在 __init__ 中定义）
pool_output = self.pool(relu_output)
# self.pool = nn.MaxPool2d(2, 2)
# 输入形状：(4, 6, 28, 28)
# 输出形状：(4, 6, 14, 14)  # 高宽各减半

# 最终结果：
x = (4, 6, 14, 14)
```

ReLU 激活的物理意义：
```text
为什么需要 ReLU？

1. 引入非线性：
   ┌─────────────────────────────────────┐
   │ 如果没有激活函数                    │
   │ → 多层网络等价于单层线性变换        │
   │ → 无法学习复杂的非线性关系          │
   └─────────────────────────────────────┘

2. 稀疏激活：
   负值被置零 → 部分神经元"沉默"
   └─> 模拟生物神经元的兴奋/抑制机制
   └─> 减少计算量，提高泛化能力

3. 梯度传播：
   对于正数区域：梯度 = 1（完美传递）
   对于负数区域：梯度 = 0（完全阻断）
   
   这使得网络能够选择性地更新某些神经元
```

### nn.functional vs nn.Module 的选择
代码中的使用模式
```python
class Net(nn.Module):
    def __init__(self):
        super().__init__()
        # 有状态的层（需要学习参数）
        self.conv1 = nn.Conv2d(3, 6, 5)  # ✓ 必须用 nn.Module
        self.fc1 = nn.Linear(400, 120)   # ✓ 必须用 nn.Module
        
        # 无状态的操作（不需要学习参数）
        # self.relu = nn.ReLU()          # ✗ 没必要，占内存
        # self.pool = nn.MaxPool2d(2,2)  # ✗ 其实也可以直接用 F.max_pool2d
    
    def forward(self, x):
        # 使用 F.xxx 直接调用（简洁）
        x = F.relu(self.conv1(x))        # ✓ 推荐
        x = F.max_pool2d(x, 2, 2)        # ✓ 也可以这样写
        x = F.relu(self.fc1(x))          # ✓ 推荐
```

选择原则：
场景          推荐方式                    原因
有参数的层   nn.Conv2d, nn.Linear    需要管理权重和偏置
激活函数        F.relu                无参数，简洁高效
池化层         两者皆可              F.max_pool2d 更简洁
Dropout      F.dropout       需要在 forward 中控制 training 模式
需要序列化     nn.Module         .state_dict() 可以保存结构

## nn.dropout 和 F.dropout
关键问题：Dropout 在不同阶段的行为
```text
# ╔══════════════════════════════════════════════════════════╗
# ║  Dropout 的双重行为                                      ║
# ╠══════════════════════════════════════════════════════════╣
# ║                                                          ║
# ║  训练模式 (net.train() 或 net.mode='train')             ║
# ║  ├─> 随机丢弃神经元（例如 50%）                         ║
# ║  ├─> 保留的神经元权重放大 2 倍（1/(1-0.5)）             ║
# ║  └─> 目的：防止过拟合，增强泛化能力                     ║
# ║                                                          ║
# ║  评估模式 (net.eval() 或 net.mode='eval')               ║
# ║  ├─> 不丢弃任何神经元                                   ║
# ║  ├─> 所有神经元都参与计算                               ║
# ║  └─> 目的：使用完整网络进行预测                         ║
# ╚══════════════════════════════════════════════════════════╝
```
场景 1：使用 nn.Dropout
```python
# 定义网络
net = Net()
net.dropout = nn.Dropout(p=0.5)

# ========== 训练阶段 ==========
net.train()  # 设置 training=True
print(net.dropout.training)  # True

# 前向传播
x = torch.ones(4, 6, 28, 28)  # 全 1 张量
output = net.dropout(x)

# nn.Dropout 内部自动检测：
# if self.training:  ← 检查的是 nn.Dropout 对象的 training 属性
#     return F.dropout(x, p=0.5, training=True)

# output 中约 50% 的位置变成 0，其余位置变成 2.0
# 例如：[[[0, 2.0, 0, 2.0], [2.0, 0, 0, 2.0], ...]]

# ========== 评估阶段 ==========
net.eval()  # 设置 training=False
print(net.dropout.training)  # False

output = net.dropout(x)
# nn.Dropout 内部：
# else:
#     return x  ← 直接返回，不做任何修改

# output = x (完全不变)
```
场景 2：使用 F.dropout
```python
# 定义网络
class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 6, 5)
    
    def forward(self, x):
        x = self.conv1(x)
        
        # 手动传递 training 参数
        x = F.dropout(x, p=0.5, training=self.training)
        #                      ↑
        #         这里引用了 Net 类的 self.training
        
        return x

# ========== 训练阶段 ==========
net = Net()
net.train()  # net.training = True

x = torch.ones(4, 6, 28, 28)
output = net(x)

# forward 中：
# F.dropout(x, p=0.5, training=True)  ← 显式传递
# output 中 50% 变 0，50% 变 2.0

# ========== 评估阶段 ==========
net.eval()  # net.training = False

output = net(x)

# forward 中：
# F.dropout(x, p=0.5, training=False)  ← 显式传递
# output = x (完全不变)
```
两者本质是等价
```python
# nn.Dropout 的源码简化版：
class Dropout(nn.Module):
    def __init__(self, p=0.5):
        super().__init__()
        self.p = p
    
    def forward(self, x):
        if self.training:
            return F.dropout(x, self.p, training=True)
        else:
            return x

# 所以：
self.dropout = nn.Dropout(p=0.5)
x = self.dropout(x)

# 等价于：
x = F.dropout(x, p=0.5, training=self.training)
```

## .state_dict() 详解
### 什么是 state_dict？
定义：
```text
.state_dict() 返回一个 Python 字典（dict），包含：
- 键（keys）：网络中每个层的名称
- 值（values）：该层的权重参数（Tensor）
```
本质：
```text
╔══════════════════════════════════════════════════════════╗
║  state_dict = 神经网络所有可学习参数的"快照"             ║
║                                                          ║
║  它只保存参数值，不保存网络结构                          ║
║  就像保存游戏的存档点，下次可以读档继续                  ║
╚══════════════════════════════════════════════════════════╝
```

场景 1：训练后保存模型

```python
# └─> 包含所有权重参数的二进制数据
```
保存过程的内部流程：
```python
# Step 1: 调用 state_dict()
state = net.state_dict()
# 返回：{'conv1.weight': tensor(...), 'conv1.bias': tensor(...), ...}

# Step 2: 序列化
# PyTorch 使用 pickle 协议将字典序列化为二进制流
# 对于每个 Tensor：
#   ├─> 保存 dtype (torch.float32)
#   ├─> 保存 shape ([6, 3, 5, 5])
#   └─> 保存实际的数值数据
```

加载过程的内部流程：
```python
# Step 1: 从磁盘读取
with open('cifar10_net.pth', 'rb') as f:
    state = torch.load(f)
# 返回：之前保存的字典

# Step 2: 反序列化
# pickle 将二进制流还原为 Python 字典
# state = {'conv1.weight': tensor(...), ...}

# Step 3: 加载到网络
net.load_state_dict(state)
# 内部执行：
# for name, param in state.items():
#     setattr(self, name, param)
# 将每个 Tensor 复制到对应的层中

# Step 4: 验证
# 检查所有参数的形状是否匹配
# 如果 conv1.weight 应该是 (6,3,5,5)，但加载的是 (10,3,5,5)，会报错
```

### 为什么需要分开保存结构和参数？
对比两种保存方式：
方式 1：保存整个模型（不推荐）
```python
# 保存整个模型对象（包括网络结构 + 参数）
torch.save(net, 'entire_model.pth')

# 加载
net = torch.load('entire_model.pth')
```
问题：
```text
❌ 代码耦合严重：
   - 必须在完全相同的代码环境中加载
   - 如果 Net 类的定义改变了，加载会失败
   - 使用了 pickle 序列化整个 Python 对象

❌ 不灵活：
   - 无法修改网络结构
   - 无法部分加载参数
   - 版本兼容性差

❌ 安全性问题：
   - pickle 可能执行任意代码
   - 加载他人模型有风险
```
方式 2：只保存 state_dict（推荐 ✓）
```python
# 只保存参数
torch.save(net.state_dict(), 'weights.pth')

# 加载
net = Net()  # 手动定义结构
net.load_state_dict(torch.load('weights.pth'))
```
优势：
```text
✓ 解耦：
   - 参数和结构分离
   - 可以在不同项目中重用
   - 版本兼容性好

✓ 灵活：
   - 可以修改网络结构
   - 可以部分加载（迁移学习）
   - 可以手动调整参数

✓ 高效：
   - 文件更小（只保存 Tensor 数据）
   - 加载更快
   - 内存占用少

✓ 安全：
   - 不执行任意 Python 代码
   - 只加载数值数据
```

### 实际操作示例
完整的训练 + 保存 + 加载流程：
```python
import torch
import torch.nn as nn
import torch.optim as optim

# ========== 1. 定义网络 ==========
class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 6, 5)
        self.fc1 = nn.Linear(400, 120)
        self.fc2 = nn.Linear(120, 10)
    
    def forward(self, x):
        x = self.conv1(x)
        x = F.relu(x)
        x = x.view(-1, 400)
        x = F.relu(self.fc1(x))
        x = self.fc2(x)
        return x

# ========== 2. 训练网络 ==========
net = Net()
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(net.parameters(), lr=0.001)

for epoch in range(10):
    for inputs, labels in trainloader:
        outputs = net(inputs)
        loss = criterion(outputs, labels)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f'Epoch {epoch+1}, Loss: {loss.item():.4f}')

# ========== 3. 保存模型 ==========
# 方式 A：只保存模型参数
torch.save(net.state_dict(), 'model_weights.pth')

# 方式 B：保存 checkpoint（包含更多信息）
checkpoint = {
    'epoch': epoch,
    'model_state_dict': net.state_dict(),
    'optimizer_state_dict': optimizer.state_dict(),
    'loss': loss.item(),
}
torch.save(checkpoint, 'checkpoint.pth')

# ========== 4. 加载模型 ==========
# 加载方式 A：只加载参数
net_new = Net()  # 必须先实例化相同结构的网络
net_new.load_state_dict(torch.load('model_weights.pth'))
net_new.eval()

# 加载方式 B：加载 checkpoint
checkpoint = torch.load('checkpoint.pth')
net_new = Net()
net_new.load_state_dict(checkpoint['model_state_dict'])
optimizer_new = optim.SGD(net_new.parameters(), lr=0.001)
optimizer_new.load_state_dict(checkpoint['optimizer_state_dict'])
start_epoch = checkpoint['epoch'] + 1
net_new.eval()
```
