# Facade数据集

CMPFacadeDatabase数据集，包含606幅手动注释的建筑物正面图像，适用于图像修复与风格迁移任务。图像来源于不同城市及建筑风格，提供图片、标签文件及详细文档。


# pix2pix 对抗学习过程

对于判别器
1. 真实对 (A, B) 送进判别器时，标签是 True
2. 假对 (A, fake_B) 送进判别器训练判别器时, 标签是 False

detach() 的作用是：训练判别器这一步时，不让梯度回传到生成器。也就是“这一步只更新 D，不更新 G”。

也就是说，训练判别器时，它要学会把生成器产出的假样本判成假。


对于生成器
errG_GAN
  - 让生成器学会“骗过判别器”
  - 不是直接学真图像素，而是学“长得像真图分布”

errG_L1
 - 让 fake_B 在像素层面接近 real_B
  - 对 pix2pix 这种配对数据尤其重要
  - 否则只靠 GAN，G 可能生成“看起来像真的”，但和输入 A 不严格对应

  所以 pix2pix 的 G 其实在同时满足两件事：

  - 要“像真的”
  - 还要“像这张输入 A 对应的那张 B”

 所以正确流程是

  1. 训练判别器 D

  - 输入真实对 (A, B)，标签 1
  - 输入假对 (A, fake_B)，标签 0
  - fake_B 要 detach()
  - 目标：让 D 能分清真假

  2. 训练生成器 G

  - 重新把 (A, fake_B) 喂给 D
  - 这次标签设成 1
  - 但此时不是训练 D，而是利用 D 的输出给 G 施压
  - 目标：让 G 生成的 fake_B 足够像真图，骗过 D


## 与DCGAN 不一样的地方
  1. DCGAN 是无条件生成

  - 输入是随机噪声 z
  - 生成器学的是“凭空造一张像真的图”

  2. pix2pix 是条件生成

  - 输入是条件图 A
  - 生成器学的是“根据 A 生成对应的 B”

## U-Net 的作用是什么
  U-Net 在这里不是为了“对抗”，而是为了“保留输入 A 的结构信息”。

  做的是：

  - 边缘图 -> 照片
  - 线稿 -> 上色图
  - 黑白图 -> 彩色图
  - 语义分割图 -> 真实街景图

  这些任务都有一个特点：
  输入 A 和输出 B 在空间结构上是强对齐的。

  例如：

  - 房子的窗户还在那个位置
  - 轮廓还在那个位置
  - 物体布局没变
  - 只是“表达方式”变了

  U-Net 的 skip connection 可以把浅层的局部结构、边缘、位置细节直接从编码器传给解码器，所以：

  - 输出更容易保留原图布局
  - 细节更清楚
  - 不容易把结构弄丢

  如果不用 U-Net，只用普通 encoder-decoder，信息一路下采样后再上采样，很多空间细节会丢。

  所以可以这样理解：

  - GAN：负责“看起来像真图”
  - U-Net：负责“和输入 A 对得上、细节别丢”
  - L1：负责“整体内容别跑偏”
