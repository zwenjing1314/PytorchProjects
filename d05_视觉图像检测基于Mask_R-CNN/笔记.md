# 数据集介绍
Penn-Fudan 行人检测和分割数据集
Penn-Fudan 行人检测和分割数据集是由宾夕法尼亚大学和复旦大学的研究者共同创建的，主要用于行人检测任务。这个数据集包含 170 张高分辨率的 RGB 图像，这些图片都是从视频序列中截取的，并且在每幅图像中有 0 到 6 个不等的行人目标。每个行人的位置都通过矩形框 (mask) 进行了精确标注，提供了边界框坐标信息，便于进行目标检测训练和测试。


## 数据集结构
数据集的文件夹结构如下：
```text
PennFudanPed/
├── Annotation  
│   
└── PedMasks  
│   
└── PNGImages

```

Annotation  注释信息
PedMasks 掩码图片（png）
PNGImages 原始图片RGB（png）

# 遇到的问题
## torch.utils.data.Dataset 的作用
它是 PyTorch 约定好的数据集接口。你写一个自己的类继承它，并实现：
__len__：一共有多少条样本（这里就是多少张图）。
__getitem__(self, idx)：按索引 idx 取出第 idx 条样本。
DataLoader 会通过 for i in range(len) 反复调用 __getitem__(i) 来取数据，再帮你组 batch、多进程、打乱等。你不继承 Dataset 也可以自己写迭代逻辑，但用标准基类的好处是和 DataLoader、教程、别人代码完全一致。

## PennFudanDataset 在这个项目里的具体作用
它把 PennFudanPed 这种「一张 RGB 图 + 一张实例编码的 mask 图」整理成 Mask R-CNN / 检测训练需要的形式：
__init__	在 PNGImages / PedMasks 里排序列文件名，保证图片和 mask 一一对应。
__getitem__	读第 idx 张图和对应 mask；用 unique 得到每个实例 id；拆成多张二值实例 mask；用 masks_to_boxes 算 XYXY 框；补 labels / area / iscrowd；包成 tv_tensors；可选 transforms；最后返回 (img, target)。

## torchvision.transforms.v2
torchvision.transforms.v2 是新一代的、功能更强大的变换API，它不仅支持图像分类，还原生支持目标检测、分割等复杂任务的数据增强。而原来的 torchvision.transforms（可称为v1）主要面向图像分类

特性维度	    torchvision.transforms (v1)	        torchvision.transforms.v2 (v2)
支持的数据类型	 仅限单张图像（PIL Image或Tensor）	    图像、视频、边界框、分割掩码、关键点等
适用任务	    主要适用于图像分类任务	                适用于分类、检测、分割、视频分类、姿态估计等多种任务
API设计与输入	输入结构固定，通常只接受图像	         接口与v1完全向后兼容，同时可接受任意数量的输入（如 (img, bboxes, masks)）或任意结构（如字典）
新增功能	    功能基本稳定，不再添加新特性	         包含所有v1功能，并增加了CutMix、MixUp、Large Scale Jitter等更先进的增强方法
性能与后端	    同时支持PIL和Tensor，但功能分散	         性能更优，尤其推荐使用Tensor后端和torch.uint8数据类型以获得最佳性能
未来方向	    不再添加新功能	                       所有新功能和改进将只在此模块中添加，是未来发展的方向

关键优势与迁移指南
v2的关键优势

    任务扩展：最核心的优势是能同时处理图像及其对应的边界框、掩码等，保证数据增强时所有元素都做一致的变换。

    功能更全：集成了更多前沿的数据增强技术。

    设计更优：输入更灵活，性能更好。

何从v1迁移到v2

迁移过程非常简单，v2的设计目标是完全向后兼容。

    第一步：更改导入语句
    只需将代码中的 import torchvision.transforms as T 改为 import torchvision.transforms.v2 as T。对于简单的图像分类任务，原有代码无需任何其他修改即可运行。

    第二步（可选）：利用v2新特性
    如果涉及检测或分割任务，或想使用CutMix等新功能，可以参考以下标准用法：

```python
import torchvision.transforms.v2 as T
from torchvision.transforms.v2 import functional as F

# 定义一个同时处理图像和边界框的变换管道
transforms = T.Compose([
    T.RandomResizedCrop(size=(224, 224)),
    T.RandomHorizontalFlip(p=0.5),
    T.ToImageTensor(),  # 将PIL图像转换为Tensor
    T.ConvertImageDtype(torch.float32),
    T.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]),
])

# 假设从数据集中获取了图像和对应的边界框
# image: PIL Image
# boxes: 边界框坐标张量
# labels: 标签张量
transformed_image, transformed_boxes, transformed_labels = transforms(image, boxes, labels)
```

版本与兼容性注意事项

    版本要求：torchvision.transforms.v2 从 0.15.0 版本开始引入，并在 0.17.0 版本中正式稳定。请确保你的 torchvision 版本不低于0.17.0。

## optimizer 和 lr_scheduler
ptimizer 负责“怎么改参数”，lr_scheduler 负责“学习率什么时候变、变成多少”

### optimizer 是干什么的
它的核心职责是：根据反向传播得到的梯度，去更新模型参数。

训练里最关键的链条是：

    前向计算得到 loss
    loss.backward() 计算每个参数的梯度
    optimizer.step() 用这些梯度更新参数

你代码里的关键位置：

清空旧梯度：optimizer.zero_grad()
反向传播：losses.backward()
参数更新：optimizer.step()

这里的 optimizer 常见有：

SGD：更传统，目标检测里很常见

    Adam
    AdamW
    它们的区别在于“更新参数的公式不同”。
比如 SGD 更像沿着梯度方向一步一步走；Adam 会自动结合一阶、二阶统计量，自适应调整每个参数的更新步幅。

你可以把它理解成：

    optimizer = “参数更新器”
    它直接作用于 model.parameters()

一个典型写法像这样：
``` python
params = [p for p in model.parameters() if p.requires_grad]
optimizer = torch.optim.SGD(params, lr=0.005, momentum=0.9, weight_decay=0.0005)
```

这里：

-   lr 是学习率
-   momentum 是动量
-   weight_decay 是权重衰减，相当于一种正则化

### lr_scheduler 是干什么的
它不直接更新模型参数。
它更新的是：optimizer 里面的学习率 lr。

也就是：

    optimizer.step() 改的是模型权重
    lr_scheduler.step() 改的是 optimizer 里的学习率配置

代码里这几行：
```python
warmup_factor = 1.0 / 1000
warmup_iters = min(1000, len(data_loader) - 1)

lr_scheduler = torch.optim.lr_scheduler.LinearLR(
    optimizer, start_factor=warmup_factor, total_iters=warmup_iters
)
```
含义就是：

-   初始学习率先从原本 lr 的 1/1000 开始
-   在前 warmup_iters 次迭代里，线性涨回正常 lr
