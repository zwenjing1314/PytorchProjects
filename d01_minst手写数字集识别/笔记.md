### 数据流向和职责
torch.utils.data 的作用
这是 PyTorch 的核心数据加载框架，提供通用的数据结构：
Dataset - 抽象类，定义了数据集的"标准接口
```python
from torch.utils.data import Dataset, DataLoader

# 1. Dataset - 定义"什么是数据集"
#    它是一个接口规范，告诉 PyTorch：
#    - 如何获取单个样本 (__getitem__)
#    - 数据集有多大 (__len__)

class CustomDataset(Dataset):
    def __init__(self, data_path):
        # 加载数据的逻辑（可以是图片、文本、音频等）
        self.data = [...]  # 任何数据
        self.labels = [...]
    
    def __getitem__(self, idx):
        # 返回第 idx 个样本
        # 数据流：索引 → 读取数据 → 返回 (sample, label)
        return self.data[idx], self.labels[idx]
    
    def __len__(self):
        # 返回数据集大小
        return len(self.data)


# 2. DataLoader - 定义"如何批量加载数据"
#    它是一个迭代器，负责：
#    - 自动 batching
#    - 数据打乱 (shuffle)
#    - 多进程加载 (num_workers)
#    - 内存管理

loader = DataLoader(
    dataset=custom_dataset,  # 任何继承 Dataset 的类
    batch_size=64,           # 每批 64 个样本
    shuffle=True,            # 每个 epoch 打乱数据
    num_workers=4            # 用 4 个子进程加载数据
)

# 数据流演示：
# DataLoader 内部调用流程：
# for batch in loader:
#     ├─> 从 Dataset 中取 64 个索引
#     ├─> 对每个索引调用 dataset[idx]
#     ├─> 将 64 个 (image, label) 打包成 batch
#     └─> 返回 (64, C, H, W), (64,)
```

torchvision.datasets 的作用
这是 PyTorch 的计算机视觉扩展库，提供现成的视觉数据集：
```python
from torchvision import datasets, transforms

# 提供了 20+ 个预实现的经典数据集
# 每个都继承了 torch.utils.data.Dataset

datasets.MNIST(...)      # MNIST 手写数字
datasets.CIFAR10(...)    # CIFAR-10 图像分类
datasets.ImageFolder(...) # 自定义文件夹图像数据集
datasets.COCO(...)       # COCO 目标检测数据集

# 以 MNIST 为例，它的内部实现类似于：
class MNIST(torch.utils.data.Dataset):
    def __init__(self, root, train=True, download=True, transform=None):
        # 1. 检查/下载数据文件
        if download:
            self._download()
        
        # 2. 解析 IDX 二进制文件
        self.data = self._parse_files()
        
        # 3. 保存预处理函数
        self.transform = transform
    
    def __getitem__(self, idx):
        # 已实现的具体逻辑：
        img = self.data[idx]           # 读取图片
        label = self.targets[idx]      # 读取标签
        
        if self.transform:
            img = self.transform(img)  # 应用预处理
        
        return img, label
    
    def __len__(self):
        return len(self.data)
```

## pt文件
### .pt文件是什么？
.pt 文件是 PyTorch 的序列化文件格式，用于保存和加载 PyTorch 对象（如张量、模型、数据集等）。
本质：
.pt = .pth = Python pickle 序列化的二进制文件
可以存储任何 Python 对象（Tensor、dict、list、自定义类等）
MNIST 的 .pt 文件存储的是：(images_tensor, labels_tensor) 的元组

### .pt 文件的内部结构
基于 PyTorch 源码和 pickle 协议，.pt 文件的结构如下：
```text
.pt 文件二进制结构：
┌─────────────────────────────────┐
│ 1. Pickle 协议头 (1-2 字节)      │  ← 标识这是 pickle 格式
│    例如：0x80 0x04 (protocol 4)  │
├─────────────────────────────────┤
│ 2. 序列化数据流                  │  ← 使用 Python 的 pickle 模块
│    ├─ 对象类型标记              │
│    ├─ Tensor 数据类型           │
│    ├─ Tensor 形状信息           │
│    ├─ 实际的数值数据            │
│    └─ 元组/列表结构信息         │
├─────────────────────────────────┤
│ 3. ZIP 容器 (PyTorch 1.6+)       │  ← 新版本用 ZIP 压缩
│    ├─ data.pkl (序列化元数据)   │
│    └─ data/ (存放实际 tensor 数据)│
└─────────────────────────────────┘
```

## torch.load的工作流程
完整的流程：
```text
import torch

# 代码调用
data, targets = torch.load('./data/MNIST/processed/training.pt')

# 内部处理流程：
"""
步骤 1: 打开文件
   └─> f = open('training.pt', 'rb')
       读取二进制内容

步骤 2: 检测文件格式
   ├─> 检查是否是 ZIP 格式 (PyTorch 1.6+)
   │   └─> 如果是：解压 ZIP 容器
   │       ├─ 提取 data.pkl (元数据)
   │       └─ 提取 data/ 文件夹 (tensor 数据)
   └─> 如果是旧版本：直接读取 pickle 流

步骤 3: 反序列化 pickle 数据
   └─> pickle.load(f)
       按照 pickle 协议解析二进制流：
       
       读取过程示例：
       0x80        → pickle 协议版本
       0x04        → protocol 4
       \x80        → TUPLE 标记
       \x95        → FRAME 标记
       ...         → 开始解析第一个 Tensor
       
       对于每个 Tensor：
       ├─ 读取类型：torch.Tensor
       ├─ 读取 dtype：torch.float32 或 torch.int64
       ├─ 读取 shape：[60000, 28, 28] 或 [60000]
       ├─ 读取 storage：实际的二进制数值
       └─ 重建 Tensor 对象

步骤 4: 重建 PyTorch 对象
   ├─ 根据 shape 分配内存
   ├─ 将二进制数据拷贝到内存
   ├─ 构建 Tensor 对象
   └─ 组装成原始结构 (tuple/list/dict)

步骤 5: 返回结果
   └─> return (images_tensor, labels_tensor)
       类型：<class 'tuple'>
       内容：(Tensor(60000,28,28), Tensor(60000))
"""
```
